# AIGC Web 模板优先创作体验 SPEC

状态：前端原型已实现并通过响应式验收  
版本：1.0  
最后更新：2026-07-11

本 SPEC 是当前用户端信息架构和创作流程的权威版本。它替代
`unified-creation-workspace-spec.md` 中“首页不进入主导航”“进入创作后直接展示编辑器”以及“所有模板统一显示高级设置”的相关决策。

## 1. 决策摘要

- 主导航固定为 `首页 / 创作 / 作品`，必须明确显示当前所在页面。
- `创作`首先是模板选择体验，不再默认直接进入上传素材和生成界面。
- 推荐模板使用 9:16 竖屏视频卡片预览；移动端左右滑动切换，视觉接近抖音内容流。
- `全部模板`使用独立页面承载分类、查找和完整列表，不塞进创作编辑器弹层。
- 用户选定模板后才进入素材上传和生成编辑器。
- 高级设置默认隐藏；只有模板明确声明需要用户调整时才显示。
- 登录头像提供常规账号菜单；未登录时点击头像进入登录流程。
- 作品、上传素材和任务历史继续统一归属 `作品` 页面。

## 2. 背景与问题

当前页面存在以下问题：

1. 点击创作后直接看到上传、设置和积分，用户尚未理解不同模板的效果。
2. 当前模板仅以一条紧凑信息展示，缺少效果预览和比较过程。
3. 模板选择器与编辑器混在一起，不适合承载完整模板发现。
4. Logo 可以返回首页，但导航没有“首页”，用户难以判断当前页面。
5. 高级设置以浮层展示，桌面端会发生裁切、错位和内容遮挡。
6. 比例、时长、分辨率和画质对多数模板并非必要决策，增加认知负担。
7. 登录头像缺少符合用户预期的账号操作。

## 3. 产品目标

- 用户进入创作后，先通过效果判断模板是否适合，再提供素材。
- 用户无需理解模型、工作流和输出参数即可完成首次生成。
- 首页、创作和作品之间具有稳定、可识别的导航关系。
- 推荐模板浏览在手机端具有视频原生、沉浸但易理解的体验。
- 完整模板库可以扩展到更多分类和模板，而不挤压编辑器。
- 普通模板使用推荐参数一键生成，复杂模板按需暴露少量设置。
- 登录用户能从头像找到常规账号管理能力。

## 4. 非目标

本轮不包含：

- 自由提示词创作或模型广场。
- 时间线、画布或专业视频编辑器。
- 模板市场、创作者入驻、购买或收藏系统。
- 个性化推荐算法。
- 团队和工作区管理。
- 允许用户修改所有底层模型参数。
- 对现有后端任务、积分和资产生命周期进行整体重构。

## 5. 信息架构

### 5.1 一级导航

桌面端：

```text
Logo     首页     创作     作品                         积分     头像
```

移动端：

```text
顶部：Logo                                      积分  头像
底部：首页                  创作                  作品
```

规则：

- 三个页面入口始终同时存在，不以 Logo 代替“首页”。
- 当前入口必须使用稳定的选中态，不只依赖悬停。
- 选中态至少同时包含背景或底色变化与文字颜色变化。
- 对应按钮使用 `aria-current="page"`。
- Logo 点击仍然返回首页，但不承担当前页面提示职责。
- 模板库不是一级导航；从首页和创作页进入。
- 任务进度不是一级导航；通过全局任务提示和作品页查看。

### 5.2 路由

| 路由 | 页面 | 主要职责 |
| --- | --- | --- |
| `#/` | 首页 | 产品价值、开始创作、推荐模板、最近动态 |
| `#/create` | 创作·选模板 | 推荐模板沉浸预览、进入全部模板 |
| `#/create?template={id}` | 创作·编辑 | 上传素材、必要确认、积分与提交 |
| `#/templates` | 全部模板 | 分类、搜索、完整模板列表与预览 |
| `#/works` | 作品 | 生成结果、进行中任务、需处理任务、上传素材 |

兼容规则：

- 旧模板深链接保留模板 ID，并进入对应模板的编辑阶段。
- 无效模板 ID 返回 `#/create`，提示“模板暂不可用，请重新选择”。
- 从作品点击“再次制作”直接进入对应模板编辑阶段。

## 6. 首页

### 6.1 页面目标

首页回答三个问题：

1. 这个产品能生成什么？
2. 我可以从哪里开始？
3. 有哪些效果值得尝试？

### 6.2 页面内容

- 首屏主标题与一句价值说明。
- 主按钮：`开始创作`，进入 `#/create`。
- 次按钮：`查看作品`；未登录或没有作品时可隐藏。
- 推荐模板区域，展示少量高质量模板效果。
- `查看全部模板`入口，进入 `#/templates`。
- 登录用户可以看到一条紧凑的最近任务或最近作品入口。

首页不得：

- 直接展示完整生成表单。
- 罗列模型、工作流、版本等内部概念。
- 同时堆叠多个同权重主按钮。

## 7. 创作页状态机

```text
进入创作
   ↓
选择模板 ─────→ 查看全部模板
   ↓                  ↓
模板效果确认 ←──── 选择模板
   ↓
上传/选择素材
   ↓
必要授权与模板专属设置
   ↓
确认积分并生成
   ↓
原地提示已提交 → 作品页查看进度或继续创作
```

### 7.1 初次进入

- 直接进入 `#/create` 时，默认显示模板选择阶段。
- 不自动选中第一张模板并跳入编辑器。
- 可以预加载第一张推荐模板的视频，但必须由用户明确点击“使用此模板”。
- 存在未完成草稿时，在模板流上方显示一个紧凑的`继续上次创作`入口；不自动覆盖模板选择页。
- 从首页模板、全部模板页或“再次制作”带模板 ID 进入时，可以直接进入编辑阶段。

### 7.2 选定模板后

- URL 写入模板 ID，支持刷新恢复和分享内部深链接。
- 页面切换到该模板的创作编辑器。
- 显示当前模板名称和`更换模板`。
- 更换模板返回模板选择阶段，并保留兼容的素材草稿。
- 输入类型不兼容时，在用户确认选择前提示需要重新选择素材。

## 8. 推荐模板沉浸预览

### 8.1 视觉模型

- 模板预览使用 9:16 竖屏视频卡片。
- 当前卡片是唯一主要焦点，相邻卡片只露出少量边缘，提示可滑动。
- 卡片内容只保留：模板名称、适用场景、时长、积分和`使用此模板`。
- 不显示工作流、模板版本、能力标签或参数列表。
- 视频不可播放时使用等比例封面，并显示明确播放按钮。

### 8.2 移动端交互

- 左右滑动切换模板，使用分页吸附；不采用上下滑动，避免与页面滚动冲突。
- 视觉参考抖音的竖屏沉浸感，但导航手势遵循本产品的左右切换约定。
- 只有当前卡片可以自动播放，默认静音并循环。
- 卡片离开主位置后必须暂停，避免多个视频同时播放。
- 点击视频切换播放/暂停；声音开关为次要图标操作。
- 点击`使用此模板`才进入编辑阶段，单纯滑动不改变草稿。
- 底部提供模板位置提示和`全部模板`入口。
- 触控目标不小于 44 × 44 像素。

### 8.3 桌面端交互

- 中央展示主要竖屏视频卡片，两侧显示上一张和下一张的局部预览。
- 支持左右按钮、触控板横向滚动和拖动切换。
- 左右方向键可以切换模板。
- 只有中央卡片播放视频。
- 页面不模拟手机外壳，不使用无意义的设备边框装饰。

### 8.4 性能与降级

- 首屏只预加载当前模板视频；相邻模板预加载封面或 metadata。
- 进入相邻模板前再加载视频资源。
- 遵循 `prefers-reduced-motion`，关闭自动播放和大幅切换动画。
- 弱网或视频失败时仍可凭封面、名称和描述完成选择。

## 9. 全部模板页

### 9.1 页面结构

```text
全部模板
找到适合商品和投放场景的成片效果

[搜索（模板数量超过 12 时显示）]
[全部] [商品展示] [投放短视频] [人像写真] ...

[模板卡] [模板卡] [模板卡]
```

### 9.2 模板卡信息

- 9:16 或模板真实比例的动态预览/封面。
- 模板名称。
- 一句适用场景。
- 时长和积分。
- 主操作：`使用模板`。
- 次操作：点击媒体预览效果。

不得显示：

- `可使用`、`已开放`等没有决策价值的标签。
- 技术工作流、版本、供应商或模型名称。
- 大段模板说明。
- 与卡片主操作重复的多个按钮。

### 9.3 选择行为

- 点击`使用模板`进入 `#/create?template={id}`。
- 返回全部模板页时保留筛选、搜索和滚动位置。
- 未发布模板不进入普通列表；需要预告时统一放在`即将上线`分组且不可提交。

## 10. 创作编辑器

### 10.1 普通模板页面结构

```text
当前模板：名称                                    更换模板

效果/素材预览                    输入素材
                                授权确认（需要时）
                                预计消耗
                                生成视频
```

- 当前模板已经在前一步确认，编辑器不再重复解释模板分类和标签。
- 没有素材时，主预览显示模板样片的柔化封面和`选择商品图/人像照片`。
- 素材选中后只保留更换、预览和移除操作，不重复显示多个“已选择”。
- 预计积分和生成按钮保持相邻。
- 提交成功不强制跳页；显示一次成功提示和`查看作品`。

### 10.2 高级设置策略

默认策略：隐藏。

- 当前所有普通模板不显示`高级设置`入口、参数摘要条或预留空白。
- 比例、时长、分辨率和画质使用模板推荐值，并写入任务快照。
- 用户不需要为默认参数确认或点击`完成设置`。
- 当前存在裁切和遮挡问题的浮动设置面板从普通流程移除。

只有满足以下条件之一的模板才允许显示设置：

- 用户选择不同平台必须改变输出比例。
- 模板业务上允许选择多个有效时长。
- 某个输入会实质改变模板效果，且无法由模板默认值推断。
- 肖像模板需要额外授权或风格选择。

显示规则：

- 模板必须明确声明可编辑字段；不能由前端按工作流类型猜测。
- 入口文案优先使用具体含义，如`选择发布平台`或`选择写真风格`，避免笼统的`高级设置`。
- 只显示该模板需要的字段，不展示全套通用参数。
- 桌面端使用编辑器内联区域，不使用会越界的浮动 Popover。
- 移动端使用可关闭的底部面板，并正确覆盖底部导航和固定生成按钮。

### 10.3 模板配置建议

现有 `outputFields` 继续表示后端支持并写入任务快照的输出字段。新增用户可编辑字段，不混淆后台能力和前台显隐：

```ts
type TemplateConfig = {
  // existing fields...
  outputFields: OutputSettingKey[]
  userEditableOutputFields?: OutputSettingKey[] // 默认 []
  editorFields?: Array<{
    id: string
    label: string
    kind: 'single-select' | 'consent'
    required: boolean
    options?: Array<{ label: string; value: string }>
  }>
}
```

规则：

- `userEditableOutputFields` 为空时不渲染设置入口。
- 普通模板使用自身默认输出参数。
- 任务仍保存完整 `outputSettings`，不因前端隐藏而丢失可追踪性。
- 设置显隐变化必须通过模板版本发布，不直接写死在页面组件中。

## 11. 作品页

作品页继续承担：

- 近期生成结果。
- 生成中任务。
- 需处理或已退款任务。
- 用户上传素材。
- 预览、下载和再次制作。

规则：

- 顶部导航明确选中`作品`。
- 不恢复独立进度页。
- 从`再次制作`进入创作编辑阶段，并预选原模板和兼容素材。
- 完整技术状态只在任务详情中显示。

## 12. 头像与账号菜单

### 12.1 未登录

- 点击头像打开登录/注册界面。
- 登录成功后回到原页面和原草稿，不跳回首页。

### 12.2 已登录

桌面端使用头像下拉菜单，移动端使用底部面板。

建议结构：

```text
头像  昵称
账号标识/绑定方式

个人资料
积分与账单
账号与安全
通知设置
帮助与反馈

退出登录
```

规则：

- 顶部积分按钮继续作为充值和积分明细的快捷入口。
- 账号菜单中的`积分与账单`可以进入同一积分面板，不复制两套数据。
- 不在账号菜单重复放置`首页 / 创作 / 作品`。
- `退出登录`与普通条目分组，并需要防误触但不强制二次确认。
- 菜单关闭后焦点返回头像。
- 点击菜单外部、按 Esc 或完成跳转后关闭菜单。

## 13. 页面状态与恢复

- 刷新编辑阶段后恢复模板 ID、兼容素材和草稿。
- 直接进入 `#/create` 时仍先显示模板选择，不被旧草稿强制跳转。
- 草稿通过`继续上次创作`恢复。
- 登录、充值或账号设置完成后恢复原页面上下文。
- 更换到输入类型不兼容的模板时，保留旧草稿快照并清空当前不兼容素材。
- 模板下架后，已有历史任务仍可查看，但不能再次提交该版本。

## 14. 文案规则

- 使用用户任务语言：`选择模板`、`使用此模板`、`选择商品图`、`生成视频`。
- 不使用`工作流`、`能力`、`Provider`、`模板版本`等内部术语。
- 不用`高级`包装普通必需设置。
- 默认参数不要求用户确认，不显示“默认参数已推荐，可以不改”之类补偿性提示。
- 一个区域只表达一个主要状态，避免同时出现`未选择`、`等待选择`和`选择后生成`。
- 空状态同时说明当前缺少什么和下一步动作。

## 15. 可访问性

- 主导航使用语义化 `nav` 和 `aria-current="page"`。
- 模板轮播提供可读的当前序号、模板名称和切换按钮名称。
- 自动播放视频默认静音，并提供播放/暂停控制。
- 键盘可以切换模板、选择模板和离开模板预览。
- 焦点不会进入不可见的相邻模板操作。
- 弹层打开时约束焦点，关闭后恢复到触发元素。
- 不能只用颜色表达当前页面、当前模板或任务状态。

## 16. 数据与埋点

至少记录：

- `home_opened`
- `primary_nav_clicked`
- `create_template_feed_opened`
- `template_preview_started`
- `template_carousel_changed`
- `all_templates_opened`
- `template_filter_changed`
- `template_selected`
- `draft_resumed`
- `asset_selected`
- `template_editor_field_changed`
- `generation_submitted`
- `avatar_menu_opened`
- `account_menu_item_clicked`

核心漏斗：

```text
进入首页/创作 → 浏览模板 → 选择模板 → 选择素材 → 提交生成 → 查看作品
```

需要分别比较：

- 推荐模板流与全部模板页的模板选择率。
- 模板选定到素材选定的转化率。
- 普通模板与需要额外设置模板的提交率。
- 登录打断前后的草稿恢复率。

## 17. 响应式要求

### 桌面端

- 顶部显示三个主导航入口。
- 模板推荐区以中央竖屏视频为焦点，不铺满大量小卡片。
- 编辑器可使用双栏，但不得因隐藏设置留下明显空洞或占位。
- 账号菜单锚定头像且不超出视口。

### 移动端

- 底部显示`首页 / 创作 / 作品`三个等宽入口。
- 模板推荐使用横向吸附的竖屏视频卡片。
- 页面垂直滚动与模板横向滑动不能互相抢夺手势。
- 编辑阶段的生成按钮固定在底部导航上方。
- 模态底部面板打开时覆盖固定生成按钮和底部导航，避免同时可操作。
- 安全区、键盘弹出和小屏高度下不得遮挡主要操作。

## 18. 验收标准

### 导航

- [x] 桌面和移动端都同时存在首页、创作、作品。
- [x] 三个路由均能正确显示唯一选中态。
- [x] Logo 返回首页，但不替代首页按钮。
- [x] 键盘和读屏能够识别当前页面。

### 模板选择

- [x] 直接进入创作页时先展示模板选择，不直接展示上传表单。
- [x] 移动端可左右滑动切换 9:16 模板预览。
- [x] 同一时间只有当前模板视频播放。
- [x] 用户必须明确点击使用模板后才进入编辑器。
- [x] 全部模板页支持分类，并在模板超过 12 个时显示搜索。
