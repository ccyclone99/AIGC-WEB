# AIGC Web 统一创作工作区 SPEC

> 已被 [template-first-creation-experience-spec.md](./template-first-creation-experience-spec.md) 部分替代。当前实现应以后者关于三级导航、模板优先创作、独立全部模板页、高级设置显隐和账号菜单的决策为准。

状态：已实现并通过前端验收  
最后更新：2026-07-11

第二轮打磨已完成：

- 未选择素材时使用当前模板样片作为柔化预览，不再显示故障感较强的纯黑空框。
- 任务完成后保留“新作品”状态，直到用户进入作品库或打开对应任务。
- 作品库默认展示生成中与已完成作品，失败记录集中在“需处理”。
- 客户可见界面不再出现“演示版”“模拟登录”“模拟充值”等内部测试措辞。
- 已移除不再使用的独立任务页、我的页和账号页壳组件。

## 1. 决策摘要

用户侧工具导航收敛为两个入口：

- `创作`：选择模板、选择素材、调整设置、提交生成、查看当前进度和最近结果。
- `作品`：管理完整生成记录和上传素材。

以下入口不再作为一级页面：

- `首页`：保留为公开产品介绍页，通过 Logo 返回。
- `模板`：合并到创作页的模板选择器。
- `进度`：合并到创作页的任务状态区和全局任务入口。
- `我的`：素材移入作品页；积分和账号继续使用顶部入口及弹层。

桌面端顶部结构：

```text
Logo    创作    作品                         任务状态    积分    头像
```

## 2. 背景与问题

当前原型将一次连续创作拆成了多个页面：

1. 用户在模板页选择模板。
2. 跳转创作页上传素材并提交。
3. 再跳转进度页查看任务。
4. 完成后到“我的”查找素材。

主要问题：

- 页面按系统对象划分，而不是按用户任务划分。
- 更换模板会离开当前创作上下文。
- 当前任务在创作页和进度页重复出现。
- “我的”同时承载素材、积分、账号，含义宽泛。
- 顶部已有积分和账号入口，与“我的”页面重复。
- 用户完成一次生成需要理解多个产品概念和页面位置。

## 3. 目标

- 用户在同一页面完成从选模板到查看首个结果的完整流程。
- 桌面端工具导航最多显示两个文字入口。
- 提交任务后不跳页，用户可以继续创作。
- 当前任务在任意工具页面一键可见。
- 完整历史和素材有清晰、唯一的归属。
- 减少重复标题、标签、状态说明和内部术语。
- 保留现有模板、任务、素材、积分和账号能力，不削减业务状态。

## 4. 非目标

本次不包含：

- 项目、文件夹或团队协作。
- 模板市场、收藏或个性化推荐。
- 大规模批量生成任务中心。
- 自由画布或时间线编辑器。
- 后端接口和数据模型重构。
- 公共首页的完整视觉重做。

## 5. 信息架构

### 5.1 公开首页

路由：`#/`

职责：

- 说明产品价值。
- 展示代表性样片和推荐模板。
- 引导进入创作。

规则：

- 不在登录后的主导航中重复显示“首页”。
- 点击 Logo 返回首页。
- 点击首页模板直接进入创作页并选中对应模板。
- 从首页开始创作时，不再经过独立模板页。

### 5.2 创作

路由：`#/create`

职责：

- 模板选择。
- 输入素材选择或上传。
- 输出设置。
- 费用确认与提交。
- 当前任务状态。
- 最近生成结果。

创作流程：

```text
选择模板 → 选择素材 → 确认设置与积分 → 提交 → 原地查看进度 → 查看结果/继续创作
```

### 5.3 作品

路由：`#/works`

职责：

- 查看全部生成任务和结果。
- 处理失败或退款任务。
- 管理用户上传的可复用素材。
- 预览、下载、复用、归档和恢复素材。

页面内一级切换：

- `生成作品`
- `上传素材`

生成作品筛选：

- `全部`
- `生成中`
- `已完成`
- `需处理`

### 5.4 积分与账号

- 点击顶部积分显示积分弹层：余额、冻结积分、充值和明细。
- 点击头像显示账号菜单或账号弹层：登录状态、登录/注册、账号安全、退出。
- 不提供独立“我的”一级页面。

## 6. 创作页详细需求

### 6.1 页面结构

桌面端：

```text
┌ 当前模板 / 更换模板 ─────────────────────────────┐
│                                                   │
│  素材与画面预览                设置与生成操作      │
│                                                   │
├ 当前任务与最近结果（有内容时显示）───────────────┤
└───────────────────────────────────────────────────┘
```

不要求永久三栏。模板选择和任务状态默认保持紧凑，主空间优先用于素材预览与生成操作。

### 6.2 模板选择器

- 当前模板以名称和预览图显示。
- 点击“更换模板”打开抽屉；移动端使用底部面板。
- 选择模板后关闭选择器，留在 `#/create`。
- 模板少于或等于 12 个时只提供分类，不显示搜索框。
- 模板超过 12 个时才显示搜索。
- 模板卡只显示：样片、名称、适用场景、时长/比例、积分。
- 不显示“可使用”“工作流类型”等无决策价值标签。
- 未开放模板默认不进入主列表；如必须展示，统一放在“即将上线”分组且不可误触提交。

模板切换规则：

- 新旧模板输入类型兼容时，保留已选素材。
- 输入类型不兼容时，选择前提示“切换后需要重新选择素材”。
- 保留兼容的输出设置；不兼容设置恢复为新模板推荐值。
- 不丢失当前草稿，不产生页面历史跳转。

### 6.3 素材与设置

- 未选择素材时，主操作为“选择商品图”或“选择人像照片”。
- 已选择素材后，提供“更换”和“移除”，预览点击媒体本身完成。
- 高级设置默认收起，并明确说明默认值可直接使用。
- 生成按钮附近只显示本次消耗、余额不足或授权要求。
- 不重复显示模板已有的比例、时长和类型信息。

### 6.4 提交与任务状态

- 提交成功后停留在创作页。
- 任务应立即出现在页面内任务状态区。
- 同一模板、素材和设置已有进行中任务时，禁止重复提交。
- 一个任务时显示任务名称、状态、进度和“查看详情”。
- 多个任务时显示最新任务，并显示“另有 N 个生成中”。
- 任务失败时显示用户可执行的下一步：重试、更换素材或查看原因。
- 任务完成时显示结果缩略图，以及预览、下载、再次制作操作。
- 最近结果最多显示 3 个；“查看全部”进入作品页。

### 6.5 全局任务入口

- 没有进行中或待处理任务时隐藏。
- 单个任务显示简短状态，例如 `生成中 42%`。
- 多个任务显示 `N 个生成中`。
- 点击后打开轻量任务面板，不强制切换页面。
- 任务完成后发送一次完成提示；提示包含“查看作品”。
- 不使用通知入口承载营销或无关消息。

## 7. 作品页详细需求

### 7.1 生成作品

每项至少显示：

- 结果或输入缩略图。
- 作品名称。
- 最近更新时间。
- 当前状态。
- 必要时显示进度或退款状态。

主要操作：

- 生成中：查看详情。
- 已完成：预览、下载、再次制作。
- 需处理：查看原因、可重试时重试。

完整技术追踪信息继续保留在任务详情中，不直接铺在列表里。

### 7.2 上传素材

复用现有素材管理能力：

- 上传。
- 预览。
- 下载。
- 重命名。
- 分类。
- 归档和恢复。
- 用于新创作。

“用于新创作”进入 `#/create`，自动选中素材，并保留兼容的当前模板。

## 8. 文案规则

- 优先使用动作和结果语言，不使用系统结构语言。
- 页面只保留一个主标题；不再叠加“Eyebrow + 页面名 + 同义说明”。
- 不向用户展示 `工作流`、`模板版本`、`幂等`、`provider` 等内部术语。
- 状态文案统一为：`等待生成`、`生成中`、`处理成片`、`已完成`、`需处理`。
- 失败提示必须同时说明发生了什么、积分是否退回、用户下一步能做什么。
- 按钮使用明确动作：`更换模板`、`选择商品图`、`开始生成`、`查看作品`。
- 避免“点击更换”“点击查看”等把交互方式写进标签；控件本身应表达可点击性。
- 演示环境说明只在必要位置出现一次，不在各页面重复。

## 9. 响应式要求

### 桌面端

- 顶部只显示 `创作`、`作品` 两个文字导航。
- 创作主区域可使用双栏，但任务状态不得挤压主预览。
- 模板选择器采用右侧抽屉或居中面板。

### 移动端

- 底部导航仅显示 `创作`、`作品`。
- 生成按钮固定在安全区上方。
- 模板选择使用底部面板。
- 当前任务使用可折叠状态条。
- 不把桌面多栏布局直接缩小到移动端。

## 10. 状态与恢复

- 刷新创作页后恢复当前草稿、模板、素材和设置。
- 登录或充值完成后返回原创作上下文。
- 从作品页返回创作页时不得自动覆盖现有草稿；只有明确点击“再次制作”或“用于新创作”才覆盖。
- 输入素材失效时保留其他设置，并提示重新选择素材。
- 当前任务状态以服务端任务为准；本地状态只用于即时反馈。

## 11. 路由迁移

- `#/templates`：重定向到 `#/create` 并打开模板选择器。
- `#/tasks`：重定向到 `#/works?status=active`。
- `#/me`：重定向到 `#/works`。
- 旧模板深链接应尽量保留模板 ID，并进入对应创作状态。
- 旧任务深链接应进入作品页并打开对应任务详情。

## 12. 可访问性

- 模板选择器和任务面板支持键盘打开、关闭和选择。
- 打开弹层后正确移动焦点，关闭后焦点回到触发控件。
- 任务进度和完成状态通过文本及 `aria-live` 提示，不只依赖颜色。
- 所有图标按钮必须有可读名称。
- 移动端主要触控目标不小于 44 × 44 像素。

## 13. 埋点

本次至少记录：

- `creation_opened`
- `template_selector_opened`
- `template_selected`
- `asset_selected`
- `generation_submitted`
- `generation_completed`
- `generation_failed`
- `work_previewed`
- `work_downloaded`
- `create_again_clicked`

重点漏斗：

```text
进入创作 → 选定模板 → 选定素材 → 提交生成 → 完成 → 预览/下载
```

## 14. 验收标准

### 导航

- [x] 桌面端主导航只有“创作”和“作品”。
- [x] Logo 可以返回首页。
- [x] 顶部积分和头像入口可独立完成原“我的”页面的对应操作。
- [x] 旧路由能够正确重定向。

### 创作

- [x] 用户无需离开创作页即可选择或更换模板。
- [x] 兼容模板切换不会丢失已选素材。
- [x] 提交后不跳页，并立即显示任务状态。
- [x] 重复任务不能被误提交。
- [x] 任务完成后可在当前页预览结果并继续创作。
- [x] 当前页最多展示 3 个最近结果。

### 作品

- [x] 生成任务和用户上传素材都有唯一、明确的管理入口。
- [x] 生成中、完成和失败任务均可查找。
- [x] 完成作品支持预览、下载和再次制作。
- [x] 素材支持上传、复用、归档和恢复。

### 文案与移动端

- [x] 页面不显示内部技术术语或无决策价值标签。
- [x] 每页只有一个主标题。
- [x] 失败状态明确说明积分和下一步操作。
- [x] 移动端只有“创作”和“作品”两个主导航入口。
- [x] 模板选择、任务状态和生成按钮在移动端可完整操作。

## 15. 代码调整映射

- `TopBar.tsx`：主导航缩减为创作、作品；增加条件式任务状态入口。
- `navigation.ts`、`types.ts`：新增 `works`，移除独立 `templates`、`tasks`、`me` 视图并兼容旧路由。
- `StudioPage.tsx`：内置模板选择器、当前任务和最近结果。
- `TemplatesView.tsx`：拆为可复用的模板选择器，不再作为一级页面。
- `TasksView.tsx`：任务列表合并到作品页；活动任务摘要复用到全局任务面板。
- `MeView.tsx`：素材管理迁移到作品页；积分和账号保留为弹层。
- `App.tsx`：页面组合收敛为首页、创作、作品。
